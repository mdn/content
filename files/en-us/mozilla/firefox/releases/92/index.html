This article provides information about the changes in Firefox 92 that will affect developers. Firefox 92 is the current Beta version of Firefox, and will ship on September 7, 2021.

Changes for web developers

Developer Tools

HTML

Removals

CSS

  • The keywords avoid-page and avoid-column are now supported for the {{cssxref("break-inside")}} property ({{bug(1722945)}}).
  • The two-value syntax for the {{cssxref("font-size-adjust")}} property is now supported ({{bug(1720131)}}).
  • The {{cssxref("@font-face/size-adjust")}} descriptor is now supported ({{bug(1720131)}}).
  • The CSS {{cssxref("accent-color")}} property has been implemented ({{bug(1722031)}}).
  • The system-ui value is now supported for the {{cssxref("font-family")}} property ({{bug(1226042)}}).

Removals

JavaScript

  • Access to audio output devices, like speakers and headphones, is now protected by the speaker-selection feature policy ({{bug(1577199)}}).
  • {{jsxref("Object.hasOwn()")}} can be used to test whether a property was defined on an object or inherited ({{bug(1721149)}}).

Removals

HTTP

  • The default HTTP {{HTTPHeader("ACCEPT")}} header for images changed to: image/avif,image/webp,*/* (following addition of support for the AVIF image format). ({{bug(1682995)}}).

Removals

Security

Removals

APIs

  • The disabledFeatures static property has been implemented for Custom Elements ({{bug(1723396)}}).

DOM

  • You can now monitor for changes to text selections in {{HTMLElement("input")}} or {{HTMLElement("textarea")}} by listening for selectionchange events in {{domxref("HTMLInputElement.selectionchange_event", "HTMLInputElement")}} and {{domxref("HTMLTextAreaElement/selectionchange_event", "HTMLTextAreaElement")}}, respectively ({{bug(1648944)}}).

Changes for add-on developers

Removals

Other

  • Support for AVIF images is now enabled by default ({{bug(1682995)}}). This format has excellent compression and no patent restrictions ( it was developed by the Alliance for Open Media). Firefox can display still images, with colorspace support for both full and limited range colors, and image transforms for mirroring and rotation. The preference image.avif.compliance_strictness can be used to adjust the compliance strictness with the specification. Animated images are not supported.
