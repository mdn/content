Houdini is the nickname of the [CSS-TAG Houdini Task Force](https://github.com/w3c/css-houdini-drafts/wiki/), which met from 2015 to 2019, and the set of low level [CSS JavaScript APIs and proposed specification](https://drafts.css-houdini.org/) that arose from that task force, some of which have been implemented.

The goal of the Houdini Task Force was to provide developers with the ability to create CSS features without having to wait for web standards specifications to define them and without waiting for every browser to fully implement the features by extending CSS, including providing the ability to hook into the styling and layout process of a browser's rendering engine. Houdini proposals gave developers access to the [CSS Object Model](/en-US/docs/Web/API/CSS_Object_Model) ({{Glossary("CSSOM")}}), enabling developers to write code the browser can parse as CSS. The premise of the Houdini features over styling with JavaScript, is that when browsers create the CSSOM — including layout, paint, and composite processes — before applying and reapplying any style updates found in scripts, performance is improved.

Houdini APIs include the [CSS Painting API](/en-US/docs/Web/API/CSS_Painting_API), the [CSS Properties and Values API](/en-US/docs/Web/API/CSS_Properties_and_Values_API), the [CSS Typed Object Model API](/en-US/docs/Web/API/CSS_Typed_OM_API), and [Worklets](/en-US/docs/Web/API/Worklet).
