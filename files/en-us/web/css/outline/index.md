<p>The <strong><code>outline</code></strong> <a href="/en-US/docs/Web/CSS">CSS</a> <a href="/en-US/docs/Web/CSS/Shorthand_properties">shorthand property</a> set all the outline properties in a single declaration.</p>

<div>{{EmbedInteractiveExample("pages/css/outline.html")}}</div>


<h2 id="Constituent_properties">Constituent properties</h2>

<p>This property is a shorthand for the following CSS properties:</p>

<ul>
  <li>{{cssxref("outline-color")}}</li>
  <li>{{cssxref("outline-style")}}</li>
  <li>{{cssxref("outline-width")}}</li>
</ul>

<h2 id="Syntax">Syntax</h2>

<pre class="brush: css no-line-numbers">/* style */
outline: solid;

/* color | style */
outline: #f66 dashed;

/* style | width */
outline: inset thick;

/* color | style | width */
outline: green solid 3px;

/* Global values */
outline: inherit;
outline: initial;
outline: revert;
outline: unset;
</pre>

<p>The <code>outline</code> property may be specified using one, two, or three of the values listed below. The order of the values does not matter.</p>

<div class="note">
<p><strong>Note:</strong> The outline will be invisible for many elements if its style is not defined. This is because the style defaults to <code>none</code>. A notable exception is <code>input</code> elements, which are given default styling by browsers.</p>
</div>

<h3 id="Values">Values</h3>

<dl>
 <dt><code>&lt;'outline-color'&gt;</code></dt>
 <dd>Sets the color of the outline. Defaults to <code>currentcolor</code> if absent. See {{cssxref("outline-color")}}.</dd>
 <dt><code>&lt;'outline-style'&gt;</code></dt>
 <dd>Sets the style of the outline. Defaults to <code>none</code> if absent. See {{cssxref("outline-style")}}.</dd>
 <dt><code>&lt;'outline-width'&gt;</code></dt>
 <dd>Sets the thickness of the outline. Defaults to <code>medium</code> if absent. See {{cssxref("outline-width")}}.</dd>
</dl>

<h2 id="Description">Description</h2>

<p><a href="/en-US/docs/Web/CSS/border">Borders</a> and outlines are very similar. However, outlines differ from borders in the following ways:</p>

<ul>
 <li>Outlines never take up space, as they are drawn outside of an element's content.</li>
 <li>According to the spec, outlines don't have to be rectangular, although they usually are.</li>
</ul>

<p>As with all shorthand properties, any omitted sub-values will be set to their <a href="/en-US/docs/Web/CSS/initial_value">initial value</a>.</p>

<h2 id="Accessibility_concerns">Accessibility concerns</h2>

<p>Assigning <code>outline</code> a value of <code>0</code> or <code>none</code> will remove the browser's default focus style. If an element can be interacted with it must have a visible focus indicator. Provide obvious focus styling if the default focus style is removed.</p>
