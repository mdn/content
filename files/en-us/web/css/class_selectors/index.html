The CSS class selector matches elements based on the contents of their {{htmlattrxref("class")}} attribute.

/* All elements with class="spacious" */
.spacious {
  margin: 2em;
}

/* All <li> elements with class="spacious" */
li.spacious {
  margin: 2em;
}

/* All <li> elements with a class list that includes both "spacious" and "elegant" */
/* For example, class="elegant retro spacious" */
li.spacious.elegant {
  margin: 2em;
}

Syntax

.class_name { style properties }

Note that this is equivalent to the following {{Cssxref("Attribute_selectors", "attribute selector")}}:

[class~=class_name] { style properties }

Examples

CSS

.red {
  color: #f33;
}

.yellow-bg {
  background: #ffa;
}

.fancy {
  font-weight: bold;
  text-shadow: 4px 4px 3px #77f;
}

HTML

<p class="red">This paragraph has red text.</p>
<p class="red yellow-bg">This paragraph has red text and a yellow background.</p>
<p class="red fancy">This paragraph has red text and "fancy" styling.</p>
<p>This is just a regular paragraph.</p>

Naming conventions

Nouns or adjectives?

When designing a CSS it comes a moment where the names that will identify the classes need to be chosen. The choice will usually fall upon a noun or an adjective. It can be useful to reserve nouns for main classes (.class) and adjectives for sub-classes (.class.subclass) in order to keep maximum portability. In this way common adjectives can have different meanings and avoid collisions among different stylesheets, since they can be declared specifically for their main classes as class-modifiers.

For instance, imagine a CSS intended to style font, margins and paragraphs of all text containers of the elements with class "book" in a book-fashion style. In this context a "red" sub-class can be used for changing the font color to red:

CSS

.book.red { 
  color: red; 
}

HTML

<article class="red book">...</article>

Now imagine another unrelated CSS that declares the class "lighbox", intended to display any element in a fixed position as a semi-transparent layer covering the entire size of the viewport. In this context a "red" sub-class can be used for changing the layer to a more reddish tonality via background-color:

CSS

.lighbox.red { 
  background-color: rgba(33, 0, 0, .9); 
}

HTML

<figure class="red lightbox">...</figure>

The common adjective "red" has been declared by two unrelated stylesheets with different meanings (in one case it refers to the color while in the other case it refers to the background-color). Collisions however are avoided thanks to the specificity whereby the subclass has been declared.
