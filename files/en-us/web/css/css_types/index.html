CSS data types define typical values (including keywords and units) accepted by CSS properties and functions. They are a special kind of component value type.

The most commonly used types are defined in the CSS Values and Units specification. This specification also defines functional notations which allow for more complex types or processing. Other types are defined in the specifications to which they apply.

Below you will find a reference to the types that you are most likely to come across, however it is not a comprehensive reference to all types defined in every CSS specification.

Syntax

In formal CSS syntax, data types are denoted by a keyword placed between the inequality signs "<" and ">".

Textual data types

These types include keywords and identifiers as well as strings, and urls.

Pre-defined keywords
Keywords with a pre-defined meaning. For example the value of collapse for the {{cssxref("<border-collapse>")}} property.
CSS-wide keyword: initial
The value specified as the property's initial value.
CSS-wide keyword: inherit
The computed value of the property on the element's parent.
CSS-wide keyword: unset
Acts as inherit or initial depending on whether the property is inherited or not.
<custom-ident>
A user-defined identifier, for example the name assigned using the {{cssxref("grid-area")}} property. See more information on the {{cssxref("<custom-ident>")}} type.
<dashed-ident>
A <custom-ident> with the additional restriction that it must start with two dashes. For example CSS Custom Properties. See more information on the {{cssxref("<dashed-ident>")}} type.
<string>
A quoted string, such as used for a value of the {{cssxref("content")}} property. See more information on the {{cssxref("<string>")}} type.
<url>
A pointer to a resource. For example as the value of {{cssxref("background-image")}}. See more information on the {{cssxref("<url>")}} type.

Numeric data types

These data types are used to indicate quantities, indexes and positions. The majority of these are defined in the Values and Units specification, however additional types are described in other specifications where they are specific to that specification along - for example the fr unit in CSS Grid Layout.

<integer>
One or more decimal units 0 through 9. See more information on the {{cssxref("<integer>")}} type.
<number>
Real numbers which may also have a fractional components. For example 1 or 1.34. See more information on the {{cssxref("<number>")}} type.
<dimension>
A number with a unit attached to it. For example 23px or 15em. See more information on the {{cssxref("<dimension>")}} type.
<percentage>
A number with a percentage sign attached to it. For example 10%. See more information on the {{cssxref("<percentage>")}} type.
<ratio>
A ratio, written with the syntax <number> / <number>. See more information on the {{cssxref("<ratio>")}} type.
<flex>
A flexible length introduced for CSS Grid Layout, written as a <dimension> with the fr unit attached and used for grid track sizing. See more information on the {{cssxref("<flex>")}} type.

Quantities

The <dimension> type is used to specify distance and other quantities.

<length>
Lengths are a <dimension> and refer to distances. See more information on the {{cssxref("<length>")}} type.
<angle>
Angles are used in properties such as {{cssxref("<linear-gradient>")}} and are a <dimension> with one of deg, grad, rad, or turn units attached. See more information on the {{cssxref("<angle>")}} type.
<time>
Duration units are a <dimension> with a s or ms unit. See more information on the {{cssxref("<time>")}} type.
<frequency>
Frequencies are a <dimension> with a Hz or kHz unit attached. See more information on the {{cssxref("<frequency>")}} type.
<resolution>
Is a <dimension> with a unit identifier of dpi, dpcm, dppx, or x. See more information on the {{cssxref("<resolution>")}} type.

Combinations of types

Some CSS properties can take a dimension or a percentage value. In this case the percentage value will be resolved to a quantity which matches the allowable dimension. Properties which can accept a percentage in addition to a dimension will use one of the types listed below.

<length-percentage>
A property that can accept a length or a percentage as a value. See more information on the {{cssxref("<length-percentage>")}} type.
<frequency-percentage>
A property that can accept a frequency or a percentage as a value. See more information on the {{cssxref("<frequency-percentage>")}} type.
<angle-percentage>
A property that can accept an angle or a percentage as a value. See more information on the {{cssxref("<angle-percentage>")}} type.
<time-percentage>
A property that can accept a time or a percentage as a value. See more information on the {{cssxref("<time-percentage>")}} type.

Color

The CSS Color Specification defines the {{cssxref("<color>")}} data type, and other types which relate to color in CSS.

<color>
Specified as a keyword or a numerical color value. See more information on the {{cssxref("<color>")}} type.
<alpha-value>
Specifies the transparency of a color. May be a <number>, in which case 0 is fully transparent and 1 is fully opaque, or a <percentage>, in which case 0% is fully transparent and 100% fully opaque.

Images

The CSS Images Specification defines the data types which deal with images, including gradients.

<image>
A url reference to an image or a color gradient. See more information on the {{cssxref("<image>")}} type.
<color-stop-list>
A list of two or more color stops with optional transition information using a color hint.
<linear-color-stop>
A <color> and a <length-percentage> to indicate the color stop for this part of the gradient.
<linear-color-hint>
A <length-percentage> to indicate how the color interpolates.
<ending-shape>
Used for radial gradients and can have a keyword value of circle or ellipse.
<size>
Detemines the size of the radial gradient's ending shape. This accepts a value of a keyword or a <length> but not a percentage.

2D positioning

The {{cssxref("<position>")}} data type is interpreted as defined for the {{cssxref("<background-position>")}} property.

{{cssxref("<position>")}}
Defines the position of an object area. Accepts a keyword value such as top or left, or a <length-percentage>.
