--- title: webSocketUrl slug: Web/WebDriver/Capabilities/webSocketUrl tags: - Reference - WebDriver - capabilities - webSocketUrl ---

With the webSocketUrl capability set to true a WebSocket server will be started in the browser, supporting bidirectional communication by using the WebDriver BiDi protocol. This capability defaults to false, and has effect for the lifetime of the session.

When the capability is set to true the returned capabilities from the New Session command will then have the same capability but with its value replaced by the WebSocket URL of the current WebDriver session.

Example

Requesting the WebSocket URL by setting the webSocketUrl capability to true:

Request: 
POST /session HTTP/1.1
{"capabilities": {"alwaysMatch": {"webSocketUrl": true}}}
Response: 
{"value":{"sessionId":"571f206f-c3fe-794c-9218-77fa89595eb9","capabilities":{[..], "webSocketUrl":"ws://localhost:9222/session/571f206f-c3fe-794c-9218-77fa89595eb9"}}}
