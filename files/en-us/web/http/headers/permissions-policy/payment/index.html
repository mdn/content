The Permissions-Policy header was previously named Feature-Policy. As of June 2021, the {{domxref('FeaturePolicy')}} API as well as some linked-to articles and demos still use the older name.

The HTTP {{HTTPHeader("Permissions-Policy")}} header field's payment directive controls whether the current document is allowed to use the Payment Request API. When this policy is disabled, the {{DOMxRef("PaymentRequest()")}} constructor will throw a {{exception("SyntaxError")}}.

Syntax

Permissions-Policy: payment <allowlist>;
<allowlist>
A list of origins for which the feature is allowed. See Permissions-Policy.

Default policy

The payment feature's default allowlist value is 'self'.
