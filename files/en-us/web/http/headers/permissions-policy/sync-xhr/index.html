The Permissions-Policy header was previously named Feature-Policy. As of June 2021, the {{domxref('FeaturePolicy')}} API as well as some linked-to articles and demos still use the older name.

The HTTP {{HTTPHeader("Permissions-Policy")}} header sync-xhrÂ directive controls whether the current document is allowed to make synchronous {{domxref("XMLHttpRequest")}} requests.

Syntax

Permissions-Policy: sync-xhr <allowlist>;
<allowlist>
A list of origins for which the feature is allowed. See Permissions-Policy.

Default policy

By default the policy is set to *, which means synchronous requests are allowed in all frames.
