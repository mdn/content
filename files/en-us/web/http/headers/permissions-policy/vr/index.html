The WebVR API has been replaced with the WebXR Device API and is currently being removed from the web platform. Use the feature identifier {{HTTPHeader("Permissions-Policy/xr-spatial-tracking","xr-spatial-tracking")}} for WebXR Device API instead.

The Permissions-Policy header was previously named Feature-Policy. As of June 2021, the {{domxref('FeaturePolicy')}} API as well as some linked-to articles and demos still use the older name.

The HTTP {{HTTPHeader("Permissions-Policy")}} header vr directive controls whether the current document is allowed to use the WebVR API. When this policy is enabled, the {{JSxRef("Promise")}} returned by {{DOMxRef("Navigator.getVRDisplays","Navigator.getVRDisplays()")}} will reject with a {{DOMxRef("DOMException")}}.

Syntax

Permissions-Policy: vr <allowlist>;
<allowlist>
A list of origins for which the feature is allowed. See Permissions-Policy.

Default policy

The default allowlist is 'self'.
