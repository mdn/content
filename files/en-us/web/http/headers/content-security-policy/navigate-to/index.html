The HTTP {{HTTPHeader("Content-Security-Policy")}} (CSP) navigate-to directive restricts the URLs to which a document can initiate navigations by any means including {{HTMLElement("form")}} (if {{CSP("form-action")}} is not specified), {{HTMLElement("a")}}, {{DOMxRef("window.location")}}, {{DOMxRef("window.open")}}, etc. This is an enforcement on what navigations this document initiates not on what this document is allowed to navigate to.

Note: If the {{CSP("form-action")}} directive is present, the navigate-to directive will not act on navigations that are form submissions.

Examples

Meta tag configuration

<meta http-equiv="Content-Security-Policy" content="navigate-to 'none'">

Violation case

Using a {{HTMLElement("form")}} element with an action set to inline JavaScript will result in a CSP violation.

<meta http-equiv="Content-Security-Policy" content="navigate-to 'none'">

<form action="javascript:alert('Foo')" id="form1" method="post">
  <input type="text" name="fieldName" value="fieldValue">
  <input type="submit" id="submit" value="submit">
</form>
