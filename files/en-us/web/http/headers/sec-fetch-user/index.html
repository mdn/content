The Sec-Fetch-User fetch metadata header is only sent for requests initiated by user activation, and its value will always be ?1.

Header type {{Glossary("Fetch Metadata Request Header")}}
{{Glossary("Forbidden header name")}} yes, since it has prefix Sec-
{{Glossary("CORS-safelisted request header")}}

Syntax

Sec-Fetch-User: ?1

Value

The value will always be ?1. When a request is triggered by something other than a user activation, the spec requires browsers to omit the header completely.
