--- title: Event reference slug: Web/Events tags: - Event - Overview - Reference ---

DOM Events notify code of "interesting changes" that may affect code execution. These can arise from user interactions such as using a mouse or resizing a window, changes in the state of the underlying environment (e.g. low battery or media events from the operating system), and other causes.

Each event is represented by an object that is based on the {{domxref("Event")}} interface, and may have additional custom fields and/or functions used to get additional information about what happened. A full list of the different event types is given in Events > Interfaces based on Event.

This topic provides links to many of the common e that fire events (the object documentation contains more detailed event information). It also shows the associated event types, and provides a small subset of the types of events that they fire.

Note

This page lists the most common events you'll come across on the web. If you are searching for an event that isn't listed here, try searching for its name, topic area, or associated specification on the rest of MDN.

Event type Description Documentation
Animation Events related to the Web Animation API. Used to respond to changes in animation status (e.g. when an animation starts or ends). Animation events fired on Document, Window, HTMLElement.
Clipboard Events related to the Clipboard API. Used to notify when content is cut, copied, or pasted. Events fired on Document, Element, Window.
Fullscreen Events related to the Fullscreen API. Used to notify when the transitioning between full screen and windowed modes, and also of errors occuring during this transition. Events fired on Document, Element.
Keyboard Events related to using a keyboard. Used to notify when keys are moved up, down, or just pressed. Events fired on Document, Element.
Mouse (+ Drag'n'drop, Wheel) Events related to using a computer mouse. Used to notify when the mouse is clicked, doubleclicked, up and down events, right-click, movement in and out of an element, text selection etc. Drag and Wheel events are derived from mouse events. Mouse events fired on Element, Drag events fired on Document, Wheel events fired on Document and Element.
Workers Events related to the Web Workers API. Used to respond to new messages and message sending errors. For service workers they also allow notification of other events, including push notifications, users clicking on displayed notifications, that push subscription has been invalidated, deletion of items from the content index, etc. Events fired on ServiceWorkerGlobalScope, DedicatedWorkerGlobalScope, SharedWorkerGlobalScope, WorkerGlobalScope, Worker, WorkerGlobalScope, BroadcastChannel.
