DOM Events notify code of "interesting changes" that may affect code execution. These can arise from user interactions such as using a mouse or resizing a window, changes in the state of the underlying environment (e.g. low battery or media events from the operating system), and other causes.

This topic provides links to some of the more common objects and their events (events are grouped under the object that they are fired on).

Window
animation, clipboard, connection, focus, gamepad, history, load & unload, manifest, messaging, print, promise rejection, transition, WebVR

clipboard, composition, focus, fullscreen, keyboard, mouse, touch

animation, input, pointer, transition

animation, clipboard, Drag'n'drop, fullscreen, keyboard, load and unload, pointer, selection, touch, transition

Depending on the browser/environment there may also be events for popups, tab changes, batteries, calls, sensors, smartcards, SMS messages, and many others.
