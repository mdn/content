The read-only RTCDataChannel property ordered indicates whether or not the data channel guarantees in-order delivery of messages; the default is true, which indicates that the data channel is indeed ordered. This is set when the {{domxref("RTCDataChannel")}} is created, by setting the ordered property on the RTCDataChannelInit object passed as {{domxref("RTCPeerConnection.createDataChannel()")}}'s options parameter.

Syntax

var ordered = aDataChannel.ordered;

A {{jsxref("Boolean")}} value which is true if in-order delivery is guaranteed and is otherwise false.

Example

var pc = new RTCPeerConnection();
var dc = pc.createDataChannel("my channel");

if (!dc.ordered) {
  // Handle unordered messaging
}
