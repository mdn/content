The PaymentDetailsBase dictionary is a mixin used by the {{domxref("PaymentDetailsInit")}} and {{domxref("PaymentDetailsUpdate")}} dictionaries. It is never directly used by developers and is included here only to be used as the basis for those documents.

Properties

{{domxref("PaymentDetailsBase.displayItems", "displayItems")}}{{optional_inline}}
An array of {{domxref("PaymentItem")}} objects, each describing one line item for the payment request. These represent the line items on a receipt or invoice.
{{domxref("PaymentDetailsBase.modifiers", "modifiers")}}{{optional_inline}}
An array of {{domxref("PaymentDetailsModifier")}} objects, each describing a modifier for particular payment method identifiers. For example, you can use one to adjust the total payment amount based on the selected payment method ("5% cash discount!").
{{domxref("PaymentDetailsBase.shippingOptions", "shippingOptions")}}{{optional_inline}}
An array of {{domxref("PaymentShippingOption")}} objects, each describing one available shipping option from which the user may choose.

Derived dictionaries

The following dictionaries include PaymentDetailsBase.

{{domxref("PaymentDetailsInit")}}
Provides payment information when calling the {{domxref("PaymentRequest.PaymentRequest", "PaymentRequest()")}} constructor.
{{domxref("PaymentDetailsUpdate")}}
Provides updated payment details while the payment user interface is being handled. This can be delivered to the payment interface using either {{domxref("PaymentDetailsUpdateEvent.updateWith()")}} or by returning it from the optional detailsUpdate promise provided to the {{domxref("PaymentRequest.show()")}} call that begins the user interaction.
