The orientation read-only property of the {{domxref("VRPositionState")}} interface returns the orientation of the sensor at the current {{domxref("VRPose.timestamp")}}, as a quarternion value.

The value is a {{domxref("Float32Array")}}, made up of the following values:

  • pitch — rotation around the X axis.
  • yaw — rotation around the Y axis.
  • roll — rotation around the Z axis.
  • w — the fourth dimension (usually 1).

The orientation yaw (rotation around the y axis) is relative to the initial yaw of the sensor when it was first read or the yaw of the sensor at the point that {{domxref("VRDisplay.resetPose()")}} was last called.

Syntax

var myOrientation = VRPose.orientation;

Value

A {{domxref("Float32Array")}}, or null if the VR sensor is not able to provide orientation data.

Examples

{{page("/Web/API/VRDisplay/getFrameData", "Examples")}}

Note: An orientation of { x: 0, y: 0, z: 0, w: 1 } is considered to be "forward".
