{{APIRef("URLPattern API")}} {{SeeCompatTable}}

The URLPattern() constructor returns a newly created {{domxref("URLPattern")}} object representing the url pattern defined by the parameters.

If the given pattern or base URL are not valid or syntactically correct, the JavaScript {{jsxref("TypeError")}} exception is thrown.

{{AvailableInWorkers}}

Syntax

const pattern = new URLPattern(input [, baseURL])

Parameters

input
The pattern that will be used for matching. This can either be a {{domxref("USVString")}}, or a structured object providing patterns for each URL part individually. Omitted parts in the structured object will be treated as wildcards.
baseURL {{optional_inline}}
A {{domxref("USVString")}} representing the base URL to use in cases where input is a relative pattern. If not specified, it defaults to undefined. If this parameter is provided, and input is an absolute pattern or a structured object, a {{jsxref("TypeError")}} is thrown.

Examples

// Matching a pathname
let pattern = new URLPattern("https://example.com/books/:id")
            = new URLPattern("/books/:id", "https://example.com")
            = new URLPattern({
                protocol: "https",
                hostname: "example.com",
                pathname: "/books/:id"
              })
            = new URLPattern({
                pathname: "/books/:id"
                baseURL: "https://example.com"
              });

// Match the protocol and hostname
let pattern = new URLPattern({
                protocol: "http{s}?",
                hostname: ":subdomain.example.com"
              });

// Match all possible structured parts
let pattern = new URLPattern({
                protocol: "http{s}?",
                username: ":username",
                password: ":password",
                hostname: ":subdomain.example.com",
                port: ":port(80|443)",
                pathname: "/:path",
                search: "*",
                hash: "*"
              });
