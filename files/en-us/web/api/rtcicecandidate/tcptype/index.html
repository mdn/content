{{APIRef("WebRTC")}}

The {{domxref("RTCIceCandidate")}} interface's read-only tcpType property is included on TCP candidates to provide additional details about the candidate type.

The tcpType field's value is set when the {{domxref("RTCIceCandidate.RTCIceCandidate", "RTCIceCandidate()")}} constructor is used. You can't directly set its value; instead, its value is automatically extracted from the {{domxref("RTCIceCandidateInit.candidate", "candidate")}} a-line, if it's formatted properly.

Syntax

var tcpType = RTCIceCandidate.tcpType;

Value

A {{domxref("DOMString")}} whose value is one of those defined by the {{domxref("RTCIceTcpCandidateType")}} enumerated type.

{{page("/en-US/docs/Web/API/RTCIceTcpCandidateType", "Values")}}

tcpType is null for {{Glossary("UDP")}} candidates.

Example

In this example, the candidate's {{domxref("RTCIceCandidate.protocol", "protocol")}} and tcpType are used to adjust the user interface for simultaneous-open TCP candidates.

if (candidate.protocol == "tcp" && candidate.tcpType == "so") {
    adjustForSimultaneousOpen(candidate);
}
