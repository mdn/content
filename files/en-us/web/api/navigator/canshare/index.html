The Navigator.canShare() method of the Web Share API returns true if the equivalent call to {{domxref("navigator.share()")}} would succeed.

The method can be used to feature-test if a data types is supported on a particular platform, including both the existing fields and new share data type extensions.

The Web Share API is gated by the web-share permission (and corresponding web-share permission policy). The canShare() method will return false if the permission is supported on the platform but has not been granted.

Warning: The method is flawed in that it may return true for invalid share data that specifies more than one property:

  • Unsupported fields will be ignored: if there are other supported and valid fields in the share data parameter the method will return true even though part of the data is not shareable.
  • The specification does not require that new supported fields in the data are valid. It is possible that extension fields might be dropped even if valid, and the data still be considered "sharable".

The method is recommended for feature detection. If used with a single field it will return true only if that field is supported and valid.

Syntax

navigator.canShare()
navigator.canShare(data)

Parameters

data {{optional_inline}}

An object defining the share data to test. Typically, an object with the same properties is passed to {{domxref("navigator.share()")}} if this call returns true.

All properties are optional but at least one valid data property must be specified or the method will return false.

The "base" set of options are listed below:

  • url: A {{domxref("USVString")}} representing a URL to be shared.
  • text: A {{domxref("USVString")}} representing text to be shared.
  • title: A {{domxref("USVString")}} representing the title to be shared.
  • files: An array of files to be shared.

Additional options may be supported in future. Properties that are not supported on a particular platform will be ignored (the method will return false if there are no other valid properties in the data).

Return value

Returns true if the specified data can be shared with {{domxref("Navigator.share()")}}, otherwise false.

Examples

Sending the MDN URL

The example checks whether navigator.canShare() is defined, and if so uses it to check whether navigator.share() can share the specified data.

HTML

The HTML just creates a paragraph in which to display the result of the test.

<p class="result"></p>

JavaScript

Note:The first test on !navigator.canShare is needed because the method maybe be undefined even if navigator.share() is supported.

let shareData = {
  title: 'MDN',
  text: 'Learn web development on MDN!',
  url: 'https://developer.mozilla.org',
}

const resultPara = document.querySelector('.result');

if (!navigator.canShare) {
  resultPara.textContent = 'navigator.canShare() not supported.';
}
else if (navigator.canShare(shareData)) {
  resultPara.textContent = 'navigator.canShare() supported. We can use navigator.share() to send the data.';
} else {
  resultPara.textContent = 'Specified data cannot be shared.';
}

Feature checking example

The javascript code fragment below demonstrates the logic for checking whether new share data is actually supported.

// Feature that may not be supported
let testShare = { someNewKey: 'Some kind of magic data' }

// Complex data that uses new key
const shareData = {
  title: 'MDN',
  text: 'Learn web development on MDN!',
  url: 'https://developer.mozilla.org',
  someNewKey: 'Some kind of magic data'
}

//Test just the key before sharing
if (navigator.canShare(testShare)) {
  // Use navigator.share() to share 'shareData'
} else {
  // Handle case that new data type can't be shared.
}
